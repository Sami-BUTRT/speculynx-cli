# Worklog — 2026-07-12

## Recommended CLI installation

A beta tester on a fresh Linux host encountered `python: command not found`
and `/usr/bin/python3: No module named pip` with the former package
installation approach. The public README now recommends `pipx`, which installs
Speculynx in an isolated Python environment without changing system Python
packages.

The documented Debian/Ubuntu commands are `sudo apt update`, `sudo apt install
-y pipx`, `pipx ensurepath`, `pipx install speculynx`, and `speculynx --help`.
The README notes that reopening the terminal can be necessary after
`pipx ensurepath`; `export PATH="$PATH:$HOME/.local/bin"` is an optional
immediate-session workaround, not a requirement.

Windows retains the pipx flow using `py -m pip install --user pipx`, `py -m
pipx ensurepath`, and `pipx install speculynx`. macOS guidance remains
deliberately limited to systems where pipx is already installed, so this does
not claim an unverified macOS setup. The README also documents `pipx upgrade
speculynx` and `pipx uninstall speculynx`.

Only documentation was changed; the CLI version and package behavior were not
changed. Platform-wide installation testing is not claimed.

Verification executed: `python -m unittest discover -s tests -v` passed (89
tests). No global CLI installation was performed.
